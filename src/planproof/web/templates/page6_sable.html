SABLE Engine

Structured Assessability via Belief Logic and Evidence -- the core innovation.

The Problem SABLE Solves

Every planning validation rule requires specific evidence to assess. "Is the building height under 4 metres?" needs an extracted height measurement. "Is the site in a Conservation Area?" needs spatial zone data. But extraction is imperfect -- measurements may be missing, low-confidence, or conflicting between sources.

Traditional systems ignore this uncertainty. If extraction produces a value, the system uses it; if not, it guesses. SABLE formalises the question: "Given the evidence we actually have, can we reliably assess this rule?" It computes a numeric belief score for each rule, representing how much evidence supports assessment, and a plausibility score representing the upper bound of what evidence could support.

The gap between belief and plausibility is ignorance -- evidence that is missing or too uncertain to count. When ignorance is high, SABLE withholds judgement rather than guessing.

Dempster-Shafer Theory (Simplified)

Bel
Belief
Minimum evidence supporting assessability. "We definitely have this much evidence."
Pl
Plausibility
Maximum possible support. "Evidence could support up to this much."
Pl - Bel
Ignorance
The gap between what we know and what we could know. Drives the "Not Assessable" state.

Unlike probability (where P(A) + P(not A) = 1), Dempster-Shafer theory allows evidence to support "I don't know" explicitly. A belief of 0.3 and plausibility of 0.9 means we have moderate evidence but high uncertainty -- exactly the situation where forcing a binary verdict would be dangerous.

The 4-Factor Mass Function

SABLE combines four factors to compute the mass of evidence for each rule. Each factor is a value between 0 and 1, and they multiply together to produce the overall evidence mass:

Reliability
How trustworthy is the extraction method?

LLM extraction from structured forms scores higher than VLM extraction from hand-drawn sketches. Based on extraction method and document quality.

Confidence
How confident is the extraction itself?

The extraction model's self-reported confidence score. Low confidence means the model struggled to read the value -- the evidence is weak.

Relevance
Does this evidence actually apply to this rule?

A height measurement is relevant to the height rule but not the garden depth rule. Ensures evidence is mapped to the correct rules.

Concordance
Do multiple sources agree?

If the form says "3.8m" and the drawing says "3.9m", concordance is high. If one says "3.8m" and the other says "7.2m", concordance drops and the evidence is weakened.

The Three Assessment States

ASSESSABLE

Belief exceeds threshold (default 0.6). The system has enough evidence to render a PASS or FAIL verdict with confidence.

Example: Height extracted from both form and drawing, values agree, confidence high. Verdict: PASS (3.8m < 4.0m limit).

PARTIALLY ASSESSABLE

Belief is moderate (0.3-0.6). Some evidence exists but it's insufficient for a reliable verdict. The system flags this for human review.

Example: Height found only in one low-confidence VLM extraction. Can't reliably say if height limit is met.

NOT ASSESSABLE

Belief is below minimum threshold (0.3). The evidence is too sparse or unreliable to make any determination.

Example: No height measurement found in any document. Rule cannot be evaluated -- reported as NA, not guessed as FAIL.

Formal Properties

SABLE satisfies three formally proven properties that guarantee safe behaviour:

  • Monotonicity: Adding more concordant evidence can only increase belief, never decrease it. More data always helps.
  • Boundedness: Belief is always in [0, 1] and belief <= plausibility. The system cannot produce paradoxical scores.
  • Determinism: Same inputs always produce same outputs. No randomness, no prompt-dependent variation.

SABLE Analysis

Belief Distribution Across Configurations

This violin plot shows how belief scores distribute across the five ablation configurations. The full system produces a bimodal distribution: high belief (>0.7) for rules with strong evidence, and near-zero belief for rules lacking evidence. This clean separation is what enables reliable three-state verdicts. Removing SABLE (ablation_d) collapses everything to binary, eliminating the uncertainty signal entirely.

Belief vs Plausibility

Each point is a rule-evaluation pair. Points on the diagonal have zero ignorance -- belief equals plausibility, meaning the evidence is conclusive. Points below the diagonal show uncertainty: the gap between what we know (belief) and what we could know (plausibility). Wide gaps indicate rules where more evidence would change the assessment, guiding officers to where their attention is most needed.
