What is Ablation?

An ablation study systematically removes one component at a time from a complete system, then measures what breaks. If removing component X causes metric Y to degrade, that's causal evidence that X contributes to Y. It's the gold standard for proving that each part of a system actually matters, borrowed from neuroscience where researchers lesion specific brain regions to map function.

Ablation Figures

False Violation Prevention

The full system with SABLE produces zero false violations across {{ n_evaluations }} evaluations. Removing SABLE (ablation_d) produces 93 false violations -- applicants would receive incorrect rejection notices for rules they actually comply with. This is the central result: assessability gating prevents the most damaging failure mode in planning validation.

Robustness Under Extraction Noise

Even when extraction noise is artificially increased to 30% (simulating much worse LLM/VLM performance), the full system maintains near-zero false violations. SABLE's evidence sufficiency framework naturally handles noisy input by lowering belief scores, causing uncertain rules to be classified as Not Assessable rather than incorrectly assessed.

True Violation Detection Under Noise

While false violations stay near zero, true violation detection (recall) degrades gracefully as noise increases. At 30% noise, the system catches fewer real violations because noisy evidence causes SABLE to withhold assessment. This is the designed behaviour -- it's better to miss a violation (which an officer will catch) than to fabricate one (which wastes everyone's time).

Verdict Distribution Across Configurations

The full system produces a balanced distribution: rules with strong evidence get clear PASS/FAIL verdicts, while rules with weak evidence are marked Not Assessable. Ablation_d (without SABLE) forces every rule into PASS or FAIL, eliminating the safety net. Ablation_a (without extraction) marks everything as Not Assessable since no evidence exists at all.

Component Contribution Analysis

This chart isolates each component's unique contribution. Only SABLE removal causes false violations to appear. Removing the SNKG or reconciliation has no effect on false violations for the current rule set, though these components serve architectural and future-proofing purposes. Removing extraction makes the system unable to assess anything, which is safe (zero false violations) but useless (zero automation).

Per-Rule Belief Across Configurations

This heatmap shows belief scores for each rule under each configuration. Rules with strong multi-source evidence (like height) maintain high belief across most configurations. Rules depending on single-source evidence (like Conservation Area status) drop more readily when components are removed. The pattern reveals which rules are most evidence-hungry and where extraction improvements would most increase automation rates.

Threshold Sensitivity Analysis

Precision (the rate of correct verdicts among issued verdicts) stays at 1.0 across all threshold values. This means the system never issues a wrong verdict regardless of where you set the assessability threshold. Lowering the threshold increases automation rate (more rules assessed) without introducing errors. This is a remarkably strong result -- the threshold controls how cautious the system is, not how accurate.

Oracle vs Real Extraction Matrix

A 2x2 experiment: run SABLE with perfect (oracle) extraction vs real (noisy) extraction, crossed with SABLE on vs off. Result: SABLE produces zero false violations in all four cells. Even with perfect extraction, removing SABLE creates false violations. Even with noisy extraction, keeping SABLE prevents them. The protection is structural, not dependent on extraction quality.

Key Findings

  • SABLE is the only component whose removal causes false violations (93 out of {{ n_evaluations }})

  • Threshold choice affects automation rate but not accuracy
