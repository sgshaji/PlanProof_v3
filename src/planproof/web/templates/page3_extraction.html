Document Classification (M1)

The first stage identifies what each uploaded file is. A site plan, an elevation drawing, a floor plan, or an application form each need different extraction strategies. Classification uses a combination of filename heuristics and LLM-based content analysis. Confidence scores indicate how certain the classification is.

FORM

Application forms with structured fields -- name, address, measurements, declarations

SITE PLAN

Bird's-eye view showing property boundaries, neighbouring buildings, access roads

ELEVATION

Side/front views showing building height, ridge lines, material finishes

FLOOR PLAN

Internal layout showing room dimensions, wall positions, door placements

LLM Text Extraction

For text-based documents (primarily application forms), PlanProof uses Groq-hosted LLMs to extract structured entities. The prompt is carefully designed to extract specific planning-relevant attributes: addresses, measurements, certificate types, and ownership declarations. Each extracted entity carries a confidence score and source provenance.

VLM Vision Extraction

Drawings contain measurements that text extraction cannot reach -- they are embedded in the visual layout of architectural plans. PlanProof uses GPT-4o's vision capabilities to read dimension labels, scale bars, and annotations directly from plan images. This is inherently noisier than text extraction but provides critical corroborating evidence.

Extraction Accuracy

Extraction Accuracy

The LLM extracts 88.6% of planning-relevant attributes from application forms, with 85.7% of extracted values matching ground truth exactly. The remaining errors are typically unit mismatches (e.g., "3800mm" instead of "3.8m") or partial extractions where the LLM captures a related but not identical value.

Prompt Improvement: v1 vs v2

The first version of the extraction prompt was too broad -- it extracted many irrelevant attributes alongside the relevant ones, yielding precision of just 0.30. The narrowed v2 prompt focuses on exactly the attributes needed for validation rules, improving precision to 0.72 while maintaining the same recall. This demonstrates that prompt engineering matters as much as model capability for structured extraction tasks.
