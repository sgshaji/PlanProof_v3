The Failure Mode

Binary PASS/FAIL on insufficient evidence

Traditional automated validation systems force a binary decision on every rule: either the application passes or it fails. When evidence is missing, ambiguous, or extracted incorrectly, these systems guess -- and guessing wrong produces false violations. An applicant receives a rejection notice for a rule they actually comply with.

This is the most damaging failure mode in planning validation. It wastes officer time re-checking correct applications, delays legitimate development, and erodes trust in automation.

The Thesis

"Don't guess when you don't know"

PlanProof introduces a third state: Not Assessable. When the system lacks sufficient evidence to make a reliable determination, it says so explicitly rather than forcing a potentially wrong binary verdict. This seemingly simple idea requires a formal evidence-sufficiency framework (SABLE) built on Dempster-Shafer theory to work correctly.

Traditional System vs PlanProof

Traditional Approach

  • Extract data from documents
  • Apply rule: value vs threshold
  • Force PASS or FAIL verdict
  • Missing data? Guess FAIL
  • Conflicting sources? Pick one
  • Low confidence? Ignore it
Result: 126 false violations per 264 evaluations (naive LLM)

PlanProof Approach

  • Extract data from documents (LLM + VLM)
  • Build knowledge graph (SNKG)
  • Reconcile conflicting evidence
  • Compute evidence sufficiency (SABLE)
  • Three-state verdicts: PASS / FAIL / Not Assessable
  • Confidence-aware decisions
Result: 0 false violations per 264 evaluations

What This Means in Practice

When a homeowner submits a planning application for a rear extension, PlanProof processes their site plans, elevation drawings, and application forms. It extracts measurements, checks them against permitted development rules, and -- crucially -- tells the planning officer which rules it could reliably assess and which ones need human review. The officer focuses their time on the uncertain cases rather than re-checking everything. No applicant receives a false rejection.
