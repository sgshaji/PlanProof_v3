The BCC dataset contains real planning applications with site plans and elevation drawings, but no application forms. This is a significant limitation -- forms contain the structured data (measurements, addresses, certificate types) that rules check against. Without forms, the real data is useful for testing document classification and drawing extraction, but not full end-to-end validation.

Why synthetic data?

Real planning applications don't come with ground truth labels. We don't know which rules truly pass or fail without a planning officer's assessment. Synthetic data lets us control every variable: building height, garden depth, site area, certificate type. Each test set has a ground_truth.json file with exact values, so we can measure extraction accuracy and verdict correctness precisely.

Each synthetic test set contains a generated application form (PDF), floor plans, elevation drawings, and a site plan -- all as both PDF and scanned PNG images. The ground truth records every attribute value used during generation, enabling exact comparison with extracted values.

Ground Truth Format

{
  "set_id": "SET_COMPLIANT_42000",
  "category": "compliant",
  "seed": 42000,
  "difficulty": "high",
  "values": [
    {
      "attribute": "proposed_building_height",
      "value": 3.8,
      "unit": "metres",
      "display_text": "3.8m"
    },
    {
      "attribute": "garden_depth_remaining",
      "value": 22.0,
      "unit": "metres",
      "display_text": "22.0m"
    }
  ],
  "expected_outcomes": {
    "R_HEIGHT_LIMIT": "PASS",
    "R_GARDEN_DEPTH": "PASS"
  }
}

The ground truth specifies exact numeric values for every measurable attribute, plus the expected PASS/FAIL outcome for each rule. This lets us separately evaluate extraction accuracy (did we get the right number?) and verdict correctness (did we reach the right conclusion?). The seed field ensures reproducibility -- the same seed always generates the same test set.

What's Representative, What's Not

Representative

  • Document types match real applications
  • Measurement ranges are realistic
  • Address formats follow UK conventions
  • Rules cover actual BCC validation criteria
  • Multiple difficulty levels test edge cases

Not Representative

  • Generated PDFs are cleaner than real scans
  • No handwritten annotations
  • Consistent formatting (real forms vary)
  • No multi-phase or amended applications
  • Limited to householder extensions
